01 — Що таке DzenQ

«Дякую»,
яке не можна спростувати

DzenQ перетворює кожен вияв вдячності на захищений від підробки цифровий сертифікат. Це може бути будь-який цифровий медіафайл — документ, зображення, аудіо чи відео — що містить особистість отримувача та криптографічний підпис, який скріплює все разом.

Жоден сервер, жодна компанія, жоден орган не може змінити або стерти його. Сертифікат живе на вашому пристрої — доступний для поширення, але незмінний, наче напис на камені.

  • 01.Підписано приватним ключем відправника — незаперечний доказ авторства
  • 02.Містить SHA-256 хеш кожного байту вмісту
  • 03.Верифікується офлайн — не потребує інтернету для підтвердження автентичності
  • 04.Створено пережити будь-яку платформу, компанію чи хмарний сервіс

03 — Можливості

Створено витримати
вічність

⬡

Підписи Ed25519

Галузевий стандарт еліптичної криптографії. Компактні, швидкі та математично незмінні без вашого приватного ключа.

◎

Офлайн-верифікація

Жодного запиту до сервера. Будь-хто з публічним ключем може перевірити автентичність — навіть через 50 років, навіть якщо DzenQ зникне.

▣

Будь-який тип медіа

Текст, зображення, аудіо, відео — хеш охоплює кожен байт. Додайте відеоповідомлення, і воно буде запечатано разом зі словами.

◈

Прив'язано до особистості

Сертифікати містять верифіковану особистість відправника. Отримувачі точно знають, хто висловив вдячність — жодного уособлення неможливо.

⬙

Передавані

Діліться з роботодавцями, відображайте в портфоліо, архівуйте у заповітах. Сертифікат мандрує скрізь, куди ви його несете.

⬚

Без центрального органу

Жодна компанія не може відкликати, змінити чи видалити ваш сертифікат. Верифікація — це математика, а не довіра до третіх сторін.

Бо кожне «дякую» заслуговує вижити

Слова зникають. Листи видаляються. Платформи закриваються. Людина, яка змінила ваше життя, може так ніколи й не дізнатись, яку вагу мали її вчинки.

DzenQ робить пам'ять міцною. Сертифікат, виданий сьогодні, можна буде перевірити через сто років — підписаний доказ того, що одна людина щиро подякувала іншій, і це було настільки важливо, що вона зафіксувала це криптографічно.

05 — Технологія

Децентралізована Особистість

Кожен користувач контролює ключову пару, згенеровану локально. Ваш публічний ключ — це ваша особистість: жодного імені, пошти чи облікового запису. Опційно можна прив'язати до DID для сумісності.

Жодних персональних даних ніде не зберігається. Особистість — ваша, портативна та суверенна.

Наш підхід

Локальна генерація ключової пари з опціональним резервним копіюванням на апаратний ключ та відновленням через схему Шаміра.

DzenQ Identity Root
├─ Public Key // Ed25519, 32 bytes
├─ Private Key // never leaves device
├─ DID Document // optional W3C DID
│ └─ Verification Method // JWK2020
└─ Recovery Scheme // Shamir SSS

Де Живуть Сертифікати

Сертифікати — самодостатні JSON-LD файли. Спершу вони живуть на вашому пристрої. Опційне прив'язування до IPFS або особистого сервера дає надмірність без залежності.

Принцип «локальне — перш за все»

Хмара не потрібна. Працює офлайн назавжди. Ви повністю контролюєте свої дані.

certificate-0042.dzenq
├─ @context // JSON-LD schema
├─ issuer // sender DID
├─ recipient // receiver DID
├─ payload_hash // SHA-256
└─ signature // Ed25519

Підписання Ed25519

Ed25519 забезпечує 128-бітний захист із 64-байтними підписами та підписуванням менш ніж за мілісекунду. Стандартизовано в RFC 8032, підтримується всіма основними криптографічними бібліотеками та апаратними ключами.

План міграції до постквантового захисту

Формат сертифіката резервує поле для ML-DSA / Dilithium. Інструменти міграції дозволять перепідписати наявні сертифікати, коли буде готово.

Signing Process
1. Serialise payload // canonical JSON
2. SHA-256 hash // 32 bytes
3. Ed25519 sign // private key op
4. Embed signature // base64url
5. Export .dzenq // self-contained

Верифікація без Довіри

Будь-хто з публічним ключем відправника може верифікувати сертифікат. Без інтернету. Без серверів DzenQ. Без довіри до будь-якої третьої сторони. Чиста математика.
