Under the Hood

Serious crypto,
invisible to the user

DzenQ is built on KERI — a modern, math-first identity protocol that works offline.

Why KERI?

KERI (Key Event Receipt Infrastructure) is a decentralized identity protocol designed from the ground up for offline capability, key rotation, and non-repudiation — exactly what DzenQ needs.

Unlike blockchain, KERI transactions are free and instant. Unlike traditional PKI, KERI doesn't require a central certificate authority. Every identity controls its own Key Event Log (KEL).

KERI Advantages

  • Works fully offline
  • Zero transaction fees
  • Key rotation built-in
  • Self-certifying identifiers
  • Math-based, not trust-based

Trade-offs

  • Newer, less battle-tested
  • Smaller ecosystem
  • Steeper learning curve

🏆 Our Choice

KERI's offline capability and free transactions win decisively over blockchain alternatives for a gratitude-at-scale system.

🔐 Key Event Log (KEL)

ICP Inception Event – AID born
ROT Rotation Event – new key, same identity
IXN Interaction Event – anchor gratitude
DIP Delegation Event – delegate to device

Every event is signed & chained.
Tamper = broken chain = detected.

Root + Delegated Architecture

Your identity is a hierarchy. A Root AID (controlled by multiple devices via multisig 2-of-3) governs key management. Each daily-use device gets its own Delegated AID for signing certificates.

This means: losing your phone doesn't mean losing your identity. Your laptop and backup key can revoke the phone and issue a new delegation — all cryptographically chained.

🛡️ Incident Recovery

Stolen device? Two remaining directors rotate the Root AID, the stolen device's delegation is revoked — all its future signatures instantly invalid.

Root AID (multisig 2/3)
├── 📱 Device AID 1 (phone)
│   └── signs daily thanks
├── 💻 Device AID 2 (laptop)
│   └── signs daily thanks
└── 🗝️ Device AID 3 (paper backup)
    └── emergency recovery only
Root never signs certificates directly.
It only manages keys and delegations.

Cross-Platform, Native Performance

DzenQ is built with Kotlin Multiplatform (KMP) targeting Android, iOS, web, and desktop from a single codebase. Cryptographic operations are handled by a Rust (keriox) library via native bindings — ensuring maximum speed and hardware keystore access.

KMP + Rust

  • One codebase, all platforms
  • Native keystore access
  • Rust-level crypto speed
  • Google-backed KMP

Trade-offs

  • Rust binding overhead
  • Smaller KMP ecosystem
